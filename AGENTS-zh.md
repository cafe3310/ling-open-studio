# Agent 协作指南

本文档为参与 `ling-open-studio` 项目协作的 AI Agent 定义了基础指令与规范。

## 🤖 核心指令 (Core Mandates)

### 1. 技能驱动开发
- 本项目严格使用 `doc-todo-log-loop` 技能进行任务管理和过程追踪。
- **严禁**在没有批准的计划或记录在案的 TODO 事项的情况下执行开发任务。
- 在完成一个有意义的工作块后，必须记录开发日志。

### 2. 文档规范
- 所有设计、研究和日志文档必须存放在 `docs/` 目录下。
- 文件名格式使用: `%Y-%m-%d-%H-%M-<标题>.md`。
- 在新文档开始处引用来源文档，以保持知识的可追溯性。

### 3. 工程卓越性
- **上下文优先级**: `AGENTS.md` 和 `GEMINI.md` 文件中的指令具有绝对最高优先级。
- **技术栈规范**: 遵循 Tailwind CSS 4, React 19 (Next.js 16) 和 LangGraph 模式。
- **验证机制**: 每一项变更都必须经过实证验证。对于 Bug 修复，需先复现失败场景。

## 🛠 项目背景 (Project Context)

- **目标**: 为 Ling 系列模型构建一个高性能、沉浸式的 Playground。
- **后端架构**: 采用模块化 LangGraph 实现，每个 Agent 拥有独立的子目录（如 `preprocessor/`, `weaver/`, `flow/`）。
- **前端架构**: 使用 Zustand 进行状态管理，`assistant-ui` 处理聊天交互，自定义组件实现特定领域的写作/网页画布。

## 📝 沟通规范
- Gemini 与用户使用 **中文** 进行沟通。
- 表达应简洁、直接且技术化。避免任何寒暄或废话。

---
*遵循这些规则，以确保系统完整性和高信号的协作效率。*
