# 提示词分布盘点清单 (Prompts Inventory)

## 1. 助手特定提示词 (Assistant Specific Prompts)

### 1.1 Naming Assistant (标题生成)
- **位置**: `apps/studio/src/assistants/naming/graph.ts`
- **变量**: `fullPrompt` (硬编码在 `generateTitleNode` 函数内)
- **用途**: 将对话摘要并生成 2-5 字的标题。

### 1.2 Web Architect - Initial Gen (网页初始生成)
- **位置**: `apps/studio/src/assistants/web-architect/initial-gen.ts`
- **变量**:
    - `templateB`: Idea Expander (需求拆解)
    - `templateA`: Style Director (视觉设计)
    - `templateC`: Code Generator (代码生成)
- **用途**: 网页生成的三个核心阶段。

### 1.3 Web Architect - Refine Gen (网页迭代更新)
- **位置**: `apps/studio/src/assistants/web-architect/refine-gen.ts`
- **变量**:
    - `templateD`: Editor (文件识别/读取)
    - `templateE`: Resolver (代码应用/写入)
- **用途**: 处理用户对已生成网页的反馈。

### 1.4 Web Architect - Data (视觉与技术栈定义)
- **位置**: `apps/studio/src/assistants/web-architect/prompts/`
- **文件**:
    - `designs.ts`: 不同视觉风格（Bento, Glassmorphism 等）的描述。
    - `tech-stacks.ts`: 技术栈（Next.js + Tailwind 等）的 Boilerplate 代码和约束。

## 2. 工具调用协议片段 (Tool Protocol Fragments)

- **位置**: `apps/studio/src/lib/tools/`
- **文件**:
    - `ctx-delimited.ts`: `conceptFragment`, `exampleFragment`, `returnExampleFragment`。
    - `ctx-json.ts`: (待验证，通常包含 JSON 模式定义和示例)。
    - `ctx-xml.ts`: (待验证，通常包含 XML 标签定义和示例)。
- **用途**: 指导模型如何使用特定的工具调用格式。

## 3. 动态注入与拼接逻辑 (Dynamic Injection & Stitching)

### 3.1 General Chat 注入
- **位置**: `apps/studio/src/app/api/chat/general/route.ts`
- **逻辑**: 调用 `strategy.spliceSystemPrompt` 将 UI 传入的 `baseSystemPrompt` 与工具协议片段拼接。

### 3.2 General Chat 降级逻辑
- **位置**: `apps/studio/src/assistants/general-chat/graph.ts`
- **逻辑**: 如果消息列表中没有 System Message，则将 `modelConfig.systemPrompt` 插入头部。

## 4. 其它 (Others)

- **Lucide Icons**: 部分提示词中散落着对 Lucide 图标使用方式的指导（如 `initial-gen.ts`）。
- **Tailwind Guidelines**: 散落在 `designs.ts` 和 `initial-gen.ts` 中。
