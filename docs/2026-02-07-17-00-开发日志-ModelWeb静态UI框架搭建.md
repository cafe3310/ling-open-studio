# 2026-02-07-17-00-开发日志-ModelWeb静态UI框架搭建

## 1. 背景 (Background)
根据 Model Web (网页生成) 的系统设计，我们需要搭建一个三栏布局的工作台界面。该界面需集成 Chat 协作、实时预览和生成配置三个核心模块，并保持 InclusionAI 的视觉风格。

## 2. 完成的工作 (Accomplishments)

### 2.1 组件开发 (`components/web/`)
- **`WebTab`**: 核心容器组件。
    - 集成了独立的 `AssistantRuntimeProvider`，为网页生成任务提供专门的 Runtime 环境。
    - 采用三栏水平布局。
- **`WebChat` (左侧)**: 基于 `assistant-ui` 的协作区。
    - 复用了现有的 `Thread` 组件。
    - 通过全局样式覆盖 (`aui-compact`) 优化了其在 320px 窄边栏下的展示效果。
- **`WebPreview` (中间)**: 网页预览舞台。
    - 实现了桌面 (`Monitor`) / 移动端 (`Smartphone`) 视图切换。
    - 预览窗口支持平滑的过渡动画。
    - 预置了模拟的 VFS 路径地址栏和 Loading 遮罩层。
- **`WebConfig` (右侧)**: 生成配置面板。
    - 实现了风格预设选择（网格布局）、技术栈下拉选择以及高级选项开关。
    - 视觉细节严格对齐 Demo 风格。

### 2.2 系统集成
- 修改了 `app/assistant.tsx`，将原本的 "Model Web" Placeholder 替换为正式的 `WebTab` 组件。
- 引入了 `assistant-ui` 的运行时隔离，确保网页生成的对话上下文不与主 Chat 混淆。

## 3. 技术细节
- **样式适配**: 使用了 Tailwind CSS 进行布局，并针对窄边栏聊天窗口进行了特定的 CSS 变量覆盖。
- **状态预览**: `WebPreview` 暂时使用静态 HTML 模拟展示生成效果，后续将对接 VFS 监听逻辑。

## 4. 后续计划 (Next Steps)
- **VFS Tooling**: 封装 `write_vfs_file` 等工具，让 AI 能够操作文件。
- **Graph 实现**: 开发 `WebGenGraph`，实现从需求到代码的自动化流转。
- **实时渲染**: 实现 `WebPreview` 与 VFS 的真实联动，通过 iframe 渲染生成产物。
