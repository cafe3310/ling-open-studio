# 视觉风格定义指南 (Visual Style Definition Guide)

## 1. 核心目标
本指南旨在教你如何将特定的设计意图转化为 AI 设计师（如 Google AI Studio）能够理解的专业术语。

## 2. 基于 InclusionAI 的风格分析
InclusionAI 的风格非常独特，它避开了常见的“黑底发光”科技感，转而追求一种**清洁、明亮、具有学术研究气息**的质感。

*   **底色 (Background)**: 极简的灰白色（Off-white），带有非常淡的冷色调（如 `#F8FAFC`）。
*   **字体 (Typography)**:
    *   **标题**: 优雅的**衬线体 (Serif)**，如 *Playfair Display* 或 *Libre Baskerville*。给人以权威、深思熟虑的感觉。
    *   **正文**: 现代的**无衬线体 (Sans-serif)**，如 *Inter* 或 *DM Sans*，注重易读性。
*   **插图 (Illustration)**: 纤细的**黑色线条 (Thin Black Outlines)** 配合局部**柔和渐变 (Soft Gradients)**。
*   **色彩 (Colors)**: 黑色为主，辅以柔和的紫色、蓝色和青色渐变。

## 3. 如何向 AI 描述视觉风格

要定义一套完整的风格，请从以下四个维度进行描述：

### 3.1 氛围与调性 (Vibe & Tone)
*   **关键词**: Academic (学术的), Clean (洁净), Research-oriented (研究导向), Sophisticated (精致的).
*   **描述词**: "A clean, research-focused aesthetic that uses white space to create a sense of intelligence and clarity." (一种干净、以研究为中心的审美，利用留白营造出智慧感和清晰感。)

### 3.2 色彩体系 (Color System)
*   **Primary**: 深邃的黑/灰 (`#1A1A1A`) 用于文字。
*   **Background**: 淡淡的冷白或浅灰。
*   **Accent**: 蓝紫色系渐变，仅用于高亮或图标内部，不要大面积使用。

### 3.3 组件与形状 (Components & Shapes)
*   **边框**: 极细的灰色线条 (`1px border, color: slate-200`)。
*   **圆角**: 适中的圆角 (`8px - 12px`)，不要过度圆滑。
*   **卡片**: 具有微弱阴影或仅靠边框区分的简洁卡片。

### 3.4 导航结构 (Layout)
*   **双层侧边栏**: 全局导航在最左侧（窄栏），功能导航在其右侧（稍宽）。

## 4. 给 Google AI Studio 的 Prompt 模板

你可以直接复制并填写以下内容发送给 AI：

---

**[Prompt 开始]**

我正在构建一个名为 "InclusionAI Demo" 的模型能力展示平台。我需要你为我定义一套设计系统。

**视觉参考 (基于 InclusionAI)**:
1.  **色调**: 采用浅色模式 (Light Mode)。底色为极其浅淡的冷灰色。
2.  **字体**: 标题使用高对比度的衬线体 (Serif)，正文使用简洁的无衬线体 (Sans-serif)。
3.  **视觉元素**: 使用纤细的黑色线条艺术 (Line Art)，并在关键图形内部填充柔和的紫蓝色渐变。
4.  **布局**: 采用双侧边栏布局 (Double Sidebar)。

**请为我输出以下内容**:
1.  **Tailwind CSS 配置**: 请给出 `tailwind.config.js` 中的颜色 (colors) 和字体 (fontFamily) 扩展配置。
2.  **核心组件样式**: 描述 Button (按钮), Card (卡片), Input (输入框) 在这种风格下的具体外观（背景、边框、圆角、阴影）。
3.  **页面布局建议**: 如何优雅地呈现“左中右”三栏结构，保持整体的“空灵感”和“呼吸感”。

**[Prompt 结束]**
