# 新项目启动与交接计划 (Transition Plan)

## 1. 战略调整
我们决定不直接在 `osw-studio` 源码上修改，而是**创建一个全新的 Git 仓库**，采用 **Greenfield (绿地)** 开发模式。

-   **基座**：使用 `assistant-ui` 官方 Starter 初始化。
-   **参考系**：同时参考 `osw-studio` (组件资产)、`ling-model-judge` (架构参考) 和 `osw-studio-hf` (部署参考)。
-   **目标**：构建一个基于 Assistant UI + LangGraph 的多模型演示平台 (InclusionAI Demo)。

## 2. 启动流程 (The Bootstrap)

### 2.1 初始化 (User Action)
1.  创建一个新的 GitHub 仓库 (e.g., `inclusion-ai-demo`)。
2.  使用 `assistant-ui` starter 初始化代码：
    ```bash
    npx create-assistant-ui@latest my-app
    ```
3.  在新项目中开启 Gemini。
4.  **关键操作**：将当前项目的 `worklog/` 目录下的所有设计文档复制到新项目中（或让 Gemini 读取）。

### 2.2 上下文清单 (Context to Carry)
在新环境中，Gemini 需要重点关注以下文档（请确保新环境能访问这些文件，或将其内容告知新 Gemini）：

*   **项目章程**: `docs/CHARTER.md` (需新建，定义 `doc-todo-log-loop` 模式)。
*   **特性设计**: `worklog/2026-02-05-20-32-特性设计草案.md` (定义了 Chat/Web/Write 三大模块)。
*   **视觉风格**: `worklog/2026-02-05-21-17-InclusionAI-DesignSystem.md` (定义了颜色、字体、Tailwind 配置)。
*   **技术架构**: `worklog/2026-02-05-21-35-技术选型决策-混合架构方案.md` (定义了 Assistant UI + LangGraph 架构)。
*   **HF 部署**: `worklog/2026-02-05-20-20-HF部署深度分析.md` (定义了 Dockerfile 和 Standalone 模式)。

### 2.3 权限配置
在新的 Gemini 会话中，请确保赋予以下目录的访问权限：
-   `Target Project` (新仓库)
-   `osw-studio` (源仓库，用于移植组件)
-   `osw-studio-hf` (参考 Dockerfile)
-   `ling-model-judge` (参考 LangGraph 实现)

## 3. 执行路线图 (Execution Roadmap)

### 阶段一：流水线打通 (Pipeline First)
**目标**：Code -> GitHub -> Build -> HF Space -> **Blank Page**。
1.  **工程初始化**：配置 Next.js Standalone 模式。
2.  **Docker 配置**：编写适配 HF 的 `Dockerfile` 和 `server.js` (参考 `osw-studio-hf`)。
3.  **HF 部署**：创建 HF Space，配置 Secrets，推送代码，确保能访问到默认的 Next.js 首页。

### 阶段二：基础架构搭建 (Infrastructure)
1.  **视觉集成**：配置 Tailwind CSS，引入 `InclusionAI` 设计系统 (颜色、字体)。
2.  **Layout 重构**：实现双层侧边栏布局。
3.  **LangGraph 接入**：搭建后端 API 路由 `/api/graph/[id]`。

### 阶段三：功能模块填充 (Features)
1.  **Model Chat**：实现基础对话，对接 OpenAI 兼容接口。
2.  **Model Web**：移植 `Preview` 组件，实现代码生成流。
3.  **Model Write**：移植 `Monaco`，实现后台分析与灵感推荐流。

## 4. 给新 Gemini 的留言
> "你好，我是上一任 Gemini。所有的设计决策、视觉规范和架构蓝图都已准备就绪，保存在 `worklog` 中。请优先执行【阶段一：流水线打通】，确保部署路径畅通后，再开始写业务代码。祝你好运！"
