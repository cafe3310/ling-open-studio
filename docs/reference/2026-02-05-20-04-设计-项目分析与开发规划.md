# 设计：项目分析与开发规划

## 背景
用户基于需要深入了解 `osw-studio` 及其在 Hugging Face 上的部署方案，并希望基于此经验开发一个类似的项目用于团队在 Hugging Face 上演示语言模型，提出了此任务。

## 来源文档列表
- `GEMINI.md`: 项目上下文及章程。
- `README.md`: 项目官方介绍。
- `package.json`: 依赖与脚本分析。
- `docs/SERVER_MODE.md`: 服务器模式详细文档。
- `../osw-studio-hf/Dockerfile`: HF 部署配置。

## 分析目标
1.  **架构拆解**：理解虚拟文件系统 (VFS)、AI 代理工具 (Shell/Patch/Eval) 和 UI (Monaco/Preview) 的集成方式。
2.  **部署机制**：分析 Next.js standalone 模式在 Hugging Face Docker 环境下的运行细节。
3.  **核心逻辑迁移**：识别哪些是通用的演示模型逻辑，哪些是特定于“构建网站”的逻辑，以便为团队开发新的演示项目。

## 方案建议
1.  **第一阶段：深度调研**。深入阅读 `lib/llm` 和 `lib/vfs` 目录，理解 AI 如何与浏览器环境交互。
2.  **第二阶段：TODO 拆分**。根据调研结果，制定开发新项目的具体步骤。
3.  **第三阶段：原型开发**。基于 `osw-studio` 的基础架构，剥离网站构建相关逻辑，替换为通用的 LLM 演示逻辑。
